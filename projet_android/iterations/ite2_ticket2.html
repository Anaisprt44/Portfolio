Ticket 2 : UI : Détails du restaurant

Consulter le consigne

Dans ce ticket, j’ai mis en place l’interface de l’écran « Détails du restaurant ». J’ai structuré la vue en deux parties : un ScrollView contenant toutes les informations (titre, nom du restaurant, adresse, description, types de cuisine, horaires), ainsi qu’un bouton « Retour » permettant de revenir à la liste des restaurants. Les textes affichés dans l'interface ont été déplacés dans le fichier strings.xml pour respecter les bonnes pratiques et faciliter une éventuelle traduction. Enfin, j’ai ajouté dans la classe DetailsRestoActivity l’action du bouton Retour grâce à l’appel de la méthode finish().

Tâches réalisées :
  • Création du layout avec ScrollView + bouton Retour
  • Externalisation des textes dans strings.xml
  • Implémentation du bouton Retour dans DetailsRestoActivity
  • Préparation des zones d’affichage des futures données API
