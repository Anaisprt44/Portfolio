Ticket 5 : Utiliser l’utilisateur resto_util dans l’API

Consulter la consigne

Programmes :

L’utilisateur resto_util ayant été créé lors du précédent projet d’AP, je le réutilise dans cette itération. Je n’ai donc eu qu’à modifier le nom de l’utilisateur ainsi que son mot de passe dans le fichier config.php de l’API.

Voici la partie modifiée du fichier : 

  <?php
  $host = "localhost";
  $port = "3306";  // Port MySQL de Wamp
  $dbname = "resto2";
  $username = "resto_util";
  $password = "secret";

Test fonctionnel :

Objectif : Vérifier que la connexion à la base fonctionne correctement avec le nouvel utilisateur.

1. Lancer l’application.
2. Cliquer sur le bouton « Voir Liste des Restaurants ».
Normalement, la liste des restaurants s’affiche correctement, ce qui confirme que l’API se connecte bien avec l’utilisateur resto_util.
