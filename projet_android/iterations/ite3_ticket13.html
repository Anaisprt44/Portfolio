Ticket 13 – Ajout de la confirmation de suppression d'une réservation

Branche utilisée : iteration3_ticket13. Ce ticket introduit une confirmation avant suppression d’une réservation pour sécuriser l’opération.

Fonctionnalité principale

Une méthode afficherPopupConfirmation() a été ajoutée. Cette méthode affiche une AlertDialog avec un champ de saisie (EditText), permettant de comparer le code de réservation entré par l’utilisateur avec le code réel de la réservation. La suppression via l’API n’est exécutée que si les codes correspondent.

Test fonctionnel
  1. Accéder à la liste des réservations et sélectionner une réservation.
  2. Cliquer sur le bouton Supprimer.
  3. Une popup s’ouvre, demandant de saisir le code de réservation.
  4. Si le code saisi ne correspond pas au code réel, la suppression est bloquée.
  5. Si le code est correct, la réservation est supprimée avec succès.
  6. Exemple : la réservation sélectionnée est supprimée correctement après saisie du code exact.
