Étape 2 - Projet Java

Dans cette étape, nous devions tester les classes métier existantes (Salarie et Service) en utilisant les classes de tests unitaires correspondantes situées dans le paquetage test.metier. Nous devions implémenter en Java la classe métier Categorie, ainsi que sa classe de test unitaire.

De plus, il fallait compléter la classe Salarie pour prendre en compte l’association Salarie-Categorie.

Ensuite, il fallait modifier la classe de test unitaire de Salarie pour refléter les modifications apportées à cette classe, puis exécuter à nouveau les tests.

Pour cette étape, nous avions des contraintes à respecter :

  • Chaque classe métier doit fournir au minimum un constructeur avec des paramètres pour initialiser les attributs.
  • Des accesseurs et mutateurs pour chaque attribut.
  • Une méthode toString permettant de vérifier l’état d’une instance de la classe.
Les classes métier doivent être regroupées dans le paquetage modele.metier, et les classes de test unitaire doivent être regroupées dans le paquetage test.metier.
