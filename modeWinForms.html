J’ai effectué un formulaire principal (FormMain) pour laisser à l’utilisateur le choix de la base de données à tester :

L’utilisateur peut cliquer en haut sur tâches et pourra choisir la base :

On peut voir ici les différents choix que lui offre la base de données pour le test des différentes tables. Et en cliquant sur un des tests (une des propositions) :

Grâce à cette interface, l’utilisateur pourra tester toutes les URLs des logos des artisans et afficher leur code HTPS. L’utilisateur clique sur « Tester toutes les URLS » et le test se lance sur la table. Pour cela, j’ai notamment utilisé les threads et le Webclient pour pouvoir faire marcher l’application en WinForm.

Puis lorsque le test est fini, une boite de dialogue (pop up) s’affiche pour dire que le test est fini et les combo-box et les radios s’affichent pour pouvoir filtrer la datagrid :

On peut filtrer par code et ainsi afficher : les codes erreurs, les codes corrects ou tout afficher comme ci-dessous en cliquant sur ce bouton radio :

On peut faire pareil avec les codes corrects et tout afficher. On peut également filtrer avec les combos box selon le nom des coopératives, selon le nom de l’entreprise, etc.

Si l’utilisateur sélectionne une coopérative, la seconde liste déroulante, avec les noms des entreprises, se met automatiquement à jour (en fonction de la coopérative sélectionnée) :

Cette mécanique, je l’ai répertoriée dans plusieurs Forms comme Form Interlocuteurs, Coopératives, sites coopératives, catalogue et marques. Comme par exemple avec la table interlocuteurs ci-dessous :

Je devais adapter mon code selon les cas. Certains Forms n’avaient qu’une seule comboBox et les radios à faire marcher ce qui était plus simple alors que d’autres devaient fonctionner avec 3 comboBox (listes déroulante) et des boutons radios en même temps.

J’ai également mis le bilan sur tous les Forms en haut à droite pour indiquer, après le test, le nombre de logos corrects et incorrects pour l’utilisateur.
