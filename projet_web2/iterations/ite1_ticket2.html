Architecture MVC en PHP

Le site R3st0.fr suit le patron Modèle-Vue-Contrôleur (MVC). Les contrôleurs récupèrent les données via les DAO, le modèle représente les entités, et la vue affiche les informations.

Les fichiers PHP utilisent include, require et use pour gérer les inclusions et namespaces, et la variable $fichier dans index.php détermine le contrôleur actif selon le paramètre action.

Le script testRestoDAO.php vérifie que les méthodes de RestoDAO fonctionnent correctement. Le dossier modele/dao contient les requêtes SQL, modele/metier les entités, et vue le code HTML.

Les contrôleurs appellent les DAO pour récupérer les données et les transmettent aux vues pour affichage. Cette architecture assure la séparation des responsabilités, facilite le test, la maintenance et le travail en équipe.
