Ticket 4 - Détecter une requête d'action inexistante

Consulter le consigne

L'objectif de ce ticket était de gérer les actions passées dans l'URL et de renvoyer la page correcte. La fonction controleurPrincipal() reçoit une action en paramètre, vérifie si elle existe dans le tableau des actions et renvoie le fichier PHP correspondant. Si l'action n'existe pas, un message d'erreur est ajouté et la page par défaut (accueil.php) est chargée. Cela permet de contrôler facilement la navigation selon les actions des utilisateurs.

Tâches réalisées :
  • Création d'une liste associant chaque action à un fichier PHP correspondant.
  • Implémentation de l'action par défaut defaut redirigeant vers accueil.php.
  • Ajout d'un message d'erreur pour les actions inexistantes.
  • Tests pour vérifier que les pages se chargent correctement selon l'action passée dans l'URL.
