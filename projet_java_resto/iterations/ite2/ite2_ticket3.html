Ticket 3 : Filtrage des avis par date

Accéder à la branche GitLab
Programmes :

Mise en place d’un système de filtrage des avis dans l’interface.

Ajout du composant JDateChooser dans la fenêtre FiltrerAvisDialog afin de permettre la sélection d’une date via un calendrier.

Le bouton “Rechercher” récupère les critères saisis (date, semaine, année), valide les entrées, puis ferme la fenêtre si la recherche est correcte.

Des contrôles sont ajoutés pour éviter les incohérences : impossibilité de saisir uniquement semaine ou année, et interdiction de mélanger date + semaine/année. En cas d’erreur, un message est affiché.

Les valeurs valides sont stockées et mises à disposition via des getters pour la fenêtre principale.

Dans ListeAvisPanel, le bouton “Filtrer” ouvre la fenêtre modale de filtrage, puis applique les critères via la méthode appliquerFiltre().

Cette méthode récupère tous les avis via le DAO, vide le tableau, puis filtre les résultats selon :

  • une date précise
  • ou une semaine + année
  • ou aucun filtre (affichage complet)

Elle compare les dates via Calendar pour extraire semaine et année, puis conserve uniquement les avis correspondant aux critères.

Enfin, les avis filtrés sont affichés dans le JTable avec : date, utilisateur, restaurant, commentaire et statut (visible / masqué).

Tests fonctionnels :
  • Filtrage par date (ex : 7 mars 2026) → affichage correct
  • Filtrage par autre date (ex : 11 mars 2026) → résultats adaptés
  • Filtrage par semaine + année → fonctionnement OK
  • Aucun résultat → tableau vide
  • Erreurs de saisie (champs incomplets ou incohérents) → pop-up d’erreur

Le système fonctionne correctement pour les différents cas d’utilisation, avec gestion des erreurs de saisie et filtrage dynamique des avis.
