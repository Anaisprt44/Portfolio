Ticket 5 : Démasquage d’un avis

Accéder à la branche GitLab
Programmes :

Mise en place de la fonctionnalité de démasquage des avis afin de rendre à nouveau visibles les critiques masquées.

Couche DAO :

Ajout de la méthode demasquerAvis(int idR, int idU) dans CritiqueDAO.

Cette méthode exécute une requête SQL de mise à jour : elle passe le champ masquer à false et supprime le motif associé (NULL) pour un avis identifié par le restaurant et l’utilisateur.

Elle retourne true si une ligne a été modifiée, sinon false, et utilise un PreparedStatement pour sécuriser la requête.

Tests DAO :

Des tests ont été ajoutés dans TestCritiqueDAO pour valider le fonctionnement de la méthode :

  • appel de la méthode demasquerAvis()
  • vérification du retour (succès/échec)
  • contrôle de l’état de l’avis en base (masquer = false)

Les tests confirment que l’avis passe correctement de masqué à visible.

Interface graphique :

Ajout de la fonctionnalité dans ListeAvisPanel via un bouton “Démasquer”.

Le bouton récupère l’avis sélectionné (idR, idU), appelle la DAO, puis affiche un message de confirmation ou d’erreur. La liste est ensuite automatiquement rafraîchie.

Cette fonctionnalité est réservée aux utilisateurs autorisés (manager).

Tests fonctionnels :
  • Vérification du passage de masquer = true à false en base
  • Affichage correct après démasquage dans l’interface
  • Contrôle des droits utilisateurs (modérateur / manager)
  • Confirmation via popup de réussite

Les tests montrent que le démasquage fonctionne correctement en base de données et dans l’interface graphique.

Conclusion :

Le système permet désormais de gérer complètement la visibilité des avis (masquage et démasquage), avec une cohérence entre la base de données, la DAO et l’interface utilisateur.
