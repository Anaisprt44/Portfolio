Ticket 7 : Génération et exécution d’un fichier JAR

Programmes :

Mise en place de la génération d’un fichier exécutable (.jar) afin de permettre le déploiement de l’application en dehors de l’environnement de développement.

Génération du JAR :

Le fichier exécutable est généré via l’action Clean & Build du projet dans l’IDE. Le résultat est disponible dans le dossier target sous forme de fichier .jar.

Une modification du fichier pom.xml a été effectuée afin d’intégrer le maven-assembly-plugin.

Cette configuration permet de générer un fat JAR, incluant toutes les dépendances du projet (notamment le driver MySQL) ainsi que la classe principale (lanceur.Main) dans le manifeste, garantissant ainsi une exécution autonome.

Test fonctionnel :

L’objectif est de vérifier le bon fonctionnement de l’application une fois compilée en JAR.

Après exécution du fichier, l’application se lance correctement sur l’écran de connexion.

  • Connexion avec user.manager@test.com / secret → accès à la liste des avis
  • Vérification des fonctionnalités de masquage et démasquage
  • Validation de l’enregistrement en base de données
Tests des fonctionnalités :

Les tests réalisés dans le JAR confirment le bon fonctionnement global de l’application :

  • Masquage d’un avis avec saisie de motif et mise à jour en base
  • Démasquage et remise à l’état visible
  • Affichage dynamique des avis selon les droits utilisateur

Le compte modérateur permet uniquement la consultation des avis visibles et le masquage, sans accès à la fonctionnalité de démasquage.

Conclusion :

Le fichier JAR généré est pleinement fonctionnel et permet d’exécuter l’application de manière autonome, tout en conservant l’ensemble des fonctionnalités et des règles de gestion des droits utilisateurs.
