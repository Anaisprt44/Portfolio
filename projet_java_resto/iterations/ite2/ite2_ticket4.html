Ticket 4 : Masquage d’un avis

Accéder à la branche GitLab
Programmes :

Mise en place d’une fonctionnalité de masquage des avis avec gestion des droits utilisateurs.

Partie base de données :

Ajout de deux colonnes dans la table critiquer :

  • masquer (booléen) : indique si l’avis est visible ou masqué
  • motif : enregistre la raison du masquage
Couche DAO :

Ajout de la méthode masquerAvis() dans CritiqueDAO permettant de mettre à jour un avis en base.

Elle met à jour le champ masquer = 1 et enregistre le motif via une requête SQL sécurisée (PreparedStatement). La méthode retourne true si la mise à jour est réussie, sinon false.

Tests DAO :

Création d’une classe de test TestCritiqueDao permettant de vérifier :

  • la création d’une critique (avec contrôle anti-doublon)
  • la récupération des avis (getAll / getByRestoId)
  • le masquage d’un avis avec motif

Le test confirme que la base de données est correctement mise à jour (champ masquer passé à 1).

Interface graphique :

Ajout d’un bouton Masquer dans ListeAvisPanel. Il ouvre une fenêtre MasquerAvisDialog permettant de saisir un motif.

Le système vérifie :

  • qu’un avis est bien sélectionné
  • que l’utilisateur a les droits (manager ou modérateur)
  • que le motif n’est pas vide

Une fois validé, l’avis est masqué en base et la liste est automatiquement rafraîchie.

Gestion des droits :

L’affichage des boutons et des avis est adapté selon le rôle :

  • Modérateur : peut masquer uniquement les avis visibles
  • Manager : peut voir et gérer les avis visibles et masqués

La méthode appliquerFiltre() a été mise à jour pour intégrer la visibilité des avis selon le rôle utilisateur.

Tests fonctionnels :
  • Masquage d’un avis via l’interface → mise à jour en base
  • Vérification du motif enregistré
  • Contrôle des droits utilisateur (modérateur / manager)
  • Affichage dynamique des avis masqués ou visibles

Les tests confirment le bon fonctionnement du masquage, la mise à jour de la base de données, ainsi que le respect des droits utilisateurs.
