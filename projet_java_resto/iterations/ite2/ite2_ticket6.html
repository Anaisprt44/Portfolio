Ticket 6 : Sécurisation de l’accès à la base de données

Accéder à la branche GitLab
Programmes :

Mise en place d’un système de sécurisation des identifiants de connexion à la base de données.

Stockage des paramètres :

Les informations de connexion sont stockées dans le fichier restoJdbc.properties, situé dans src/main/resources, dossier géré par Maven et intégré automatiquement au classpath.

Le mot de passe de la base de données est chiffré en AES afin d’éviter toute exposition en clair dans le projet.

Chiffrement :

Création de la classe utilitaire CryptoUtils permettant de chiffrer les mots de passe avec l’algorithme AES.

AES est un algorithme symétrique sécurisé (128, 192 ou 256 bits) utilisé en entreprise. Il est choisi pour sa robustesse et son intégration native dans Java (javax.crypto).

Une clé secrète est stockée dans un fichier .env, chargé via la bibliothèque dotenv-java.

Déchiffrement :

Ajout d’une méthode privée decrypt() dans ConnexionBDD.

Cette méthode reconstruit une clé AES, décode le mot de passe en Base64, puis le déchiffre pour obtenir la valeur en clair utilisée lors de la connexion.

Connexion à la base :

La méthode getConnexion() a été adaptée :

  • chargement des propriétés via ClassLoader
  • lecture des variables du fichier .env
  • décryptage du mot de passe
  • connexion via DriverManager

Une erreur est levée si la clé secrète est absente afin de garantir la sécurité.

Tests fonctionnels :

Un test unitaire (TestConnexionBDD) a été utilisé pour vérifier la connexion à la base de données.

Un test via l’interface graphique confirme également que la connexion utilisateur fonctionne correctement :

  • connexion avec user.moderator@test.com
  • mot de passe : secret
  • accès à la liste des avis après authentification

Le test est concluant : la connexion est sécurisée et fonctionnelle.
