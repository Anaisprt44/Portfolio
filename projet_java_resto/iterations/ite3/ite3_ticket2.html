Ticket 2 - Suppression d'un avis

Accéder à la branche GitLab
Programmes :

Dans le cadre de la suppression d’un avis, j’ai ajouté une méthode delete(int idR, int idU) dans la classe CritiqueDao. Cette méthode permet de supprimer définitivement une critique en base de données à partir de l’identifiant du restaurant et de l’utilisateur.

Une requête SQL DELETE est utilisée avec un PreparedStatement afin de sécuriser l’exécution et éviter les injections SQL. Les paramètres sont injectés avec setInt, puis la requête est exécutée avec executeUpdate().

La méthode retourne true si une ligne a été supprimée, et false sinon. En cas d’erreur SQL, une exception est capturée et la méthode retourne également false.

Des tests unitaires ont été réalisés pour valider cette méthode :

  • Création d’une critique de test en base de données
  • Suppression de cette critique avec la méthode delete()
  • Vérification que la critique n’existe plus en base

Ensuite, dans la classe ListeAvisPanel.java, j’ai ajouté la méthode supprimerAvis() permettant de supprimer un avis depuis l’interface graphique.

  • Récupération de l’avis sélectionné
  • Vérification qu’un avis est bien sélectionné
  • Affichage d’une boîte de confirmation
  • Appel de la méthode delete()
  • Actualisation de la liste des avis

Si la suppression réussit, un message de confirmation est affiché. Sinon, un message d’erreur informe l’utilisateur.

Un bouton « Supprimer » a également été ajouté dans l’interface. Ce bouton déclenche un événement qui appelle la méthode supprimerAvis(), permettant de séparer la logique métier de l’interface graphique et d’améliorer la maintenabilité du code.

Test fonctionnel :

Objectif : vérifier le bon fonctionnement de la suppression d’un avis ainsi que les droits d’accès selon le rôle utilisateur.

Tests des méthodes :

  • Création puis suppression d’une critique
  • Vérification en base de données que la critique est supprimée
  • Validation du bon fonctionnement de la méthode delete()

Test interface - Cas modérateur :

  • Connexion avec un compte modérateur
  • Accès à la liste des avis
  • Constat : le bouton « Supprimer » n’est pas visible

Test interface - Cas responsable :

  • Connexion avec un compte responsable
  • Sélection d’un avis dans la liste
  • Clic sur le bouton « Supprimer »
  • Affichage d’une boîte de confirmation
  • Validation de la suppression
  • Résultat : l’avis disparaît de la liste
  • Vérification en base : l’avis est bien supprimé

Ces tests ont permis de valider que la suppression fonctionne correctement, qu’elle est sécurisée et qu’elle respecte les droits des utilisateurs.
